## Conflict Prevention Architecture

### âŒ WRONG WAY (Causes Conflicts)

```
Component.tsx
    â†“ directly imports
mockData.ts OR supabase.ts
```

**Problem:** When switching between mock and real data, you need to edit components!  
**Result:** Conflicts between GitHub and Figma Make edits.

---

### âœ… RIGHT WAY (No Conflicts)

```
Component.tsx (Edit in Figma Make)
    â†“ imports from
dataService.ts (Never edit)
    â†“ routes to
mockData.ts (Figma) OR supabase.ts (GitHub)
```

**Solution:** Components never change! Only config flag changes.  
**Result:** No conflicts - different files being edited.
